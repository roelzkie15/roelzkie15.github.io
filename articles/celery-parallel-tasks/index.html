Celery is a distributed task queue framework that delegates tasks across multiple workers or threads. It supports parallel processing. As a reader, I assume that you already have a prior knowledge about the basics of Celery and this article will only focus on the parallel tasks feature of Celery.

The parallel capacity of Celery is based on how many worker instances you run and how many child processes in a single worker instance are set. 

                $ celery -A celery_app worker --loglevel=INFO

Notice that this single worker has 8 child processors by default. See concurrency: 8 (prefork).

This means that my machine can do 8 processes at the same time. It could be different in your case. Here's how you check the CPU details in the macOS-based machine. 

                $ sysctl -n hw.physicalcpu

For Linux-based OS see this link: https://unix.stackexchange.com/q/218074/278825.

You can also control the number of concurrency by providing --concurrency or -c option when starting a celery worker. 

                $ celery -A celery_app worker --loglevel=INFO -c 2

You may also check the pool details of the celery_app via: 

                $ celery -A celery_app inspect stats

In the pool information, check the processes. 

                
"pool": {
    ... 
    "processes": [
        15491,
        15492
    ]
}

Now it's only limited to processing 2 tasks simultaneously.

Now let's explore celery parallel tasks execution with the given worker setup above having 2 concurrencies.

Imagine you have a function cpu_intensive_func that processes something that could take seconds to complete.

from celery import Celery, group

    """Consumes CPU resources for the specified duration and intensity."""

    message = f"Execution ID: {id} / Process ID: {os.getpid()}"
    start_time = time.time()

        for _ in range(intensity):
            # Intensive floating-point calculation
            x = 1.00000001 ** 10000000

    return message


# Rollout 10 cpu_intensive_func tasks each will take 3 seconds.
group(cpu_intensive_func.s(i, 3, 100000) for i in range(10))()

Notice the Process ID: [PID]. Each cpu_intensive_func is delegated in a separate worker threads (15491 and 15492) and the processes took 12-15 seconds to complete.

If we only set a single worker with one processor (concurrency): 

                $ celery -A celery_app worker --loglevel=INFO -c 1

The tasks will take 30 seconds to complete and this is not parallelism anymore.

Conclusion

So Celery parallelism is based on the number of concurrency and increasing the concurrency will decrease the task completion time. But make sure to only set concurrency not more than the number of your CPU cores otherwise, it will cause some performance issues by taking other CPU resources and switching over again which is resource-consuming and may affect other applications.
